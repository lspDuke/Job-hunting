# 智能招聘搜索技能 (Job Search Agent)

根据用户提供的所在地、目标岗位或简历信息，自动搜索网上各大招聘平台的最新职位信息，并整理成结构化表格反馈。

## 功能特性

- ✅ **用户无需提供 API Key**：API Key 从 OpenClaw 环境变量继承（使用已配置的 `TAVILY_API_KEY`）
- ✅ 用户只需要提供求职信息：工作地点 + 目标岗位
- 支持根据用户所在地 + 目标岗位关键词搜索
- 支持从简历中提取用户期望信息后搜索
- 自动整理成表格输出（Markdown 表格），字段见下方“输出格式”
- 支持排除特定类型岗位（如标注类）
- 支持设置薪资要求和其他筛选条件
- 输出同时打印到控制台；如指定 `--output` 则额外写入文件（UTF-8，覆盖写入）

## 使用方式

### 在 OpenClaw 对话中使用

当用户提供以下信息后，即可调用：
1. **用户所在地（工作地点）** ✅ 必填
2. **目标岗位/关键词** ✅ 必填
3. （可选）最低薪资要求
4. （可选）需要排除的岗位关键词

### 命令行调用

```bash
node {baseDir}/scripts/search-wrapper.mjs --location "重庆" --keywords "AI 智能体 AI Agent AI训练师" --min-salary 6000 --exclude "标注员 数据标注"
```

### 参数说明

| 参数 | 必填 | 说明 | 示例 |
|------|------|------|------|
| `--location` | ✅ | 工作地点 | `重庆`、`北京` |
| `--keywords` | ✅ | 搜索关键词（岗位名称） | `AI 智能体 AI Agent` |
| `--min-salary` | ⬜ | 最低薪资要求（元） | `6000` |
| `--exclude` | ⬜ | 排除关键词（一个字符串；脚本会按空格拆分为多个关键词） | `标注员 数据标注` |
| `--count` | ⬜ | 返回结果数量（默认 10；实际抓取会先取最多 20 条候选再筛选） | `5` |
| `--output` | ⬜ | 输出到 Markdown 文件 | `result.md` |

### 输出格式

返回 Markdown 表格，包含以下字段：

| 字段 | 说明 |
|------|------|
| 公司名称 | 招聘公司名称 |
| 公司地址 | 公司所在地址（工作地点） |
| 注册资金 | 公司注册资本 |
| 实缴资金 | 公司实缴资本（如果能搜索到） |
| 公司评价 | 公司评价/推荐理由 |
| 匹配度 | 岗位匹配度评分（满分 100 分） |
| 招聘网站 | 信息来源的招聘网站 |
| 岗位链接 | 原始岗位链接 |

**输出字段的可靠性约定（避免编造/幻觉数据）：**

- **注册资金 / 实缴资金**：仅在从搜索结果摘要中能解析到明确的字段时填写；否则填 `未知`。该字段仅供参考，建议以企业信息公示渠道/权威企业信息站点为准。
- **薪资筛选**：只有在岗位摘要中能解析出薪资数字时才参与 `--min-salary` 过滤；对“面议/不标薪/无法解析”的岗位，会视为“无法判断”，默认不会因为薪资原因被过滤。
- **未知值统一规则**：脚本当前使用 `未知`/`暂无`/`无` 等中文占位（而非 `Unknown`），下游如需机器解析请按这些占位值处理。

**匹配度（0-100）计算说明（与脚本一致）：**

- **基础规则**：最终分数会被限制在 **10–100** 区间（即最低不会低于 10 分）。
- **公司规模（注册资金）贡献**（仅当注册资金不是 `未知` 且能解析到数字时）：
  - 注册资金数值 \(\ge 1000\)：+30
  - 注册资金数值 \(\ge 100\)：+20
  - 注册资金数值 \< 100：+10
- **关键词命中贡献**：将 `--keywords` 按空格拆分为关键词列表；每命中 1 个关键词（在岗位摘要中出现）+15。
- **推荐理由**：会把“公司规模判断 + 关键词命中 +（如有）公开评分/好评/差评倾向”等组合成一句中文推荐理由，作为“公司评价”字段输出的一部分。

**薪资解析支持的格式（与脚本一致）：**

- 支持从摘要中识别类似：`10K-20K`、`10K/月`、`月薪 10`（按脚本现有正则提取；会将提取到的数字视为 K 并换算为元）
- 不支持/不稳定的示例：`8-12k·13薪`、`20-40万/年`、`25-35k*14`、`面议` 等（这些通常会被视为“无法判断”，不会因薪资被过滤）

**示例表格：**

| 公司名称 | 公司地址 | 注册资金 | 实缴资金 | 公司评价 | 匹配度 | 招聘网站 | 岗位链接 |
|----------|----------|----------|----------|----------|--------|----------|----------|
| 某某科技有限公司 | 重庆 | 500万 | 100万 | 人工智能领域初创公司，提供良好成长空间 | 85分 | BOSS直聘 | [链接](https://...) |

## 工作流程

1. 接收用户输入（所在地 + 岗位关键词）
2. 构建搜索查询，组合岗位关键词与常见招聘站点关键字（例如在查询中加入站点/品牌词），以便从公开网页搜索中更快命中相关结果
3. 调用 Tavily Search API 获取搜索结果（使用已配置的 API Key）
4. 解析搜索结果，提取公司信息
5. 二次搜索补充公司注册/实缴资金信息（仅在可可靠匹配到公司主体时填写，无法确认则不填）
6. 根据筛选条件（薪资、排除关键词）过滤结果
7. 整理成 Markdown 表格格式输出

## 配置

- 使用 **Tavily Search** 进行搜索，**API Key 从 OpenClaw 环境变量继承**
- **用户无需额外提供 API Key**，只需要提供求职相关信息（地点 + 岗位关键词）
- 运行环境需要配置 `TAVILY_API_KEY`；未配置时应报错并提示如何设置该环境变量

## 退出码与错误行为（便于自动化/定时任务）

- 缺少 `TAVILY_API_KEY`：退出码 1
- 缺少必填参数 `--location` / `--keywords`：退出码 1
- 调用搜索服务失败或发生未捕获错误：退出码 1
- 未找到结果或筛选后为空：正常退出（退出码 0），并输出提示信息

## 注意事项（重要）

- 本技能通过搜索引擎聚合公开网页信息并做摘要整理，**不保证实时性、完整性与准确性**；请以岗位原链接与企业官方/权威渠道为准。
- 不同网站的摘要格式差异较大，薪资与工商字段可能缺失或无法稳定解析；建议将结果作为“线索清单”，再进行人工核对与投递。
