# job-search-agent（智能招聘搜索技能）

根据用户提供的**工作地点**与**目标岗位关键词**，通过搜索引擎聚合公开网页招聘信息，并输出结构化的 Markdown 表格（可用于快速生成“线索清单”）。

## 适用场景

- 快速搜集某城市某岗位的最新招聘线索
- 设置最低薪资、排除关键词（如过滤“标注/外包”等）
- 将结果输出到文件，方便二次整理/分享

## 运行环境与前置条件

- **Node.js**（项目为 ESM：`"type": "module"`）
- 环境变量中需要配置：`TAVILY_API_KEY`
  - 未配置会直接报错并以退出码 1 退出

## 安装依赖

```bash
npm install
```

## 命令行使用

脚本入口：`scripts/search-wrapper.mjs`

```bash
node ./scripts/search-wrapper.mjs --location "重庆" --keywords "AI 智能体 AI Agent AI训练师" --min-salary 6000 --exclude "标注员 数据标注"
```

### 参数说明

| 参数 | 必填 | 说明 |
|------|------|------|
| `--location` | ✅ | 工作地点（城市/地区） |
| `--keywords` | ✅ | 岗位关键词（用空格分隔多个关键词） |
| `--min-salary` | ⬜ | 最低薪资要求（元）。只有当摘要中能解析出薪资数字时才会参与过滤 |
| `--exclude` | ⬜ | 排除关键词（传一个字符串；脚本会按空格拆分为多个关键词） |
| `--count` | ⬜ | 返回结果数量（默认 10；实际会先取最多 20 条候选再筛选） |
| `--output` | ⬜ | 输出到 Markdown 文件（UTF-8，覆盖写入） |

## 输出说明

- 默认会将结果**打印到控制台**
- 若指定 `--output result.md`，会**额外写入文件**
- 输出为 Markdown 表格，字段如下：
  - 公司名称、公司地址、注册资金、实缴资金、公司评价、匹配度、招聘网站、岗位链接

### 字段可靠性与占位值

- **注册资金/实缴资金**：仅在搜索结果摘要中能解析到明确字段时填写；否则为 `未知`（仅供参考，以权威渠道为准）。
- **薪资**：不同网站摘要格式差异大；对“面议/不标薪/无法解析”的岗位，默认不会因薪资原因被过滤。
- **占位值**：脚本当前使用 `未知` / `暂无` / `无` 等中文占位值。

## 匹配度说明（与脚本一致）

- 最终分数会被限制在 **10–100** 区间
- 公司规模（注册资金）贡献（仅当注册资金可解析且非 `未知`）：
  - \(\ge 1000\)：+30
  - \(\ge 100\)：+20
  - \< 100：+10
- 关键词命中贡献：`--keywords` 按空格拆分后，每命中 1 个关键词（在岗位摘要中出现）+15

## 退出码与错误行为（便于定时任务）

- 缺少 `TAVILY_API_KEY`：退出码 1
- 缺少必填参数 `--location` / `--keywords`：退出码 1
- 调用搜索服务失败或发生未捕获错误：退出码 1
- 未找到结果或筛选后为空：正常退出（退出码 0），并输出提示信息

## 定时任务示例

```bash
# 每天早上 8:00 搜索重庆 AI 相关岗位（示例）
0 8 * * * node /path/to/job-search-agent/scripts/search-wrapper.mjs --location "重庆" --keywords "AI 智能体 AI Agent AI训练师" --min-salary 6000 --exclude "标注员 数据标注" | send-to-feishu
```

## 注意事项（重要）

- 本项目通过搜索引擎聚合公开网页信息并做摘要整理，**不保证实时性、完整性与准确性**；请以岗位原链接与企业官方/权威渠道为准。
- 结果更适合作为“线索清单”，建议人工核对后再投递。
